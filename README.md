# CS 320 Portfolio Artifact

Software Testing, Automation, and Quality Assurance

This repository contains artifacts from CS 320 demonstrating my understanding of software testing, automation, and quality assurance. The included projects showcase my ability to develop software that meets requirements, create and execute unit tests, identify defects, and evaluate software quality.

How can I ensure that my code, program, or software is functional and secure?

I can ensure that my software is functional and secure by following software development best practices, validating input data, and creating comprehensive unit tests. During this course, I used JUnit testing to verify that program requirements were met and to identify defects before deployment. Testing helps confirm that software behaves as expected while reducing the risk of errors and vulnerabilities.

How do I interpret user needs and incorporate them into a program?

I interpret user needs by carefully reviewing requirements and identifying the features and behaviors users expect from the software. Once requirements are understood, I design and implement solutions that satisfy those needs while ensuring reliability and usability. Throughout this course, I translated project requirements into classes, methods, and tests that validated the expected functionality.

How do I approach designing software?

I approach software design by first analyzing requirements and breaking problems into smaller, manageable components. I then create classes and methods that each serve a specific purpose while maintaining organization and readability. Testing is incorporated throughout development to ensure that each component functions correctly and that the overall system meets user requirements.
